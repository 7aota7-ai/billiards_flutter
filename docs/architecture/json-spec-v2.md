# JSON仕様 v2.0

## 位置付け

本ドキュメントは、ビリヤード競技力向上システムにおけるJSONデータ仕様を定義する。

JSONは以下の共通データ形式とする。

```text
ChatGPT
  ↓ JSON
system_v2.gs
  ↓
Google Spreadsheet

将来:
Flutter
  ↓ JSON
GAS API / Backend
  ↓
Google Spreadsheet または DB
```

---

## 現在フェーズ

現在はGAS再構築フェーズである。  
既存GASを修正し続けるのではなく、現行Spreadsheet構造と本JSON仕様を基に `system_v2.gs` を新規作成する。

---

## バージョン方針

- 現行仕様は `schema_version: "2.0"` とする。
- v2.0では `mist_logs` の誤記修正（→`miss_logs`）、`practice_records`・`variation_map_records` のJSONスキーマからの削除（Manual運用へ移行）、`pro_reference_records` のUpsert化を行った。
- 旧 `1.x` JSONはsystem_v2.gsの互換対応対象としない。必要な場合は個別に移行する。
- v2.0以降、既存キー・既存IDの意味は変更しない。
- 新しい項目やシートを追加する場合は、既存JSONを壊さない形で追加する。
- 既存キー名の変更・削除が必要な場合はマイナーバージョンを上げる（例：v2.1）。

---

## 基本方針

### 1. JSONキー名はSpreadsheet列名と一致させる

GASはシート1行目のヘッダーとJSONキー名を照合して書き込む。  
列番号には依存しない。

### 2. 実シート名を使用する

現行実シート名を正とする。

特に以下に注意。

```text
誤：練習設計
正：メニュー設計
```

### 3. 数式列・入力規則を壊さない

GASは以下を行わない。

- 入力規則削除
- 数式列上書き
- シート全体クリア
- ヘッダーの勝手な変更

## run

`run` は、GASでどの更新処理を実行するかを指定する。

- `true`：処理する
- `false`：処理しない
- キーが存在しない場合は `false` 扱い

## training_records

対象シート：練習ログ

日々の練習実施ログを記録する。

### 項目

|項目|必須|内容|

|日付|必須|実施日|
|カテゴリ|任意|センターショット、Position Lab等|
|内容|必須|実施内容|

### 振り返り行（DEC00016）

その日の`総評`・`良かったこと`・`課題`・`振り返り観点`・`明日以降の練習方針`は、列を追加せず、その日最後の1行として以下の形で追加する。

|項目|値|
|---|---|
|日付|対象日|
|カテゴリ|`振り返り`|
|内容|総評・良かったこと・課題をまとめた文章|
|気付き|振り返り観点・明日以降の練習方針をまとめた文章|

`today_focus`にはこれらの項目を含めない（次項参照）。

## position_lab_records

対象シート：Position Lab  
更新方式：Append

### 記入タイミングの2区分

項目は「記録時（練習直後）」と「Review時（見返し時）」の2区分で運用する。  
シート列・JSONキーは変更しない。区分は記入タイミングの運用ルールであり、スキーマ変更ではない。

- **記録時**：その場で観測しないと後から復元できない事実・実測データ。練習直後はこの区分のみ埋めればよい。
- **Review時**：後から考えても情報が劣化しない解釈・判断。見返し時に「気付き（生メモ）」から抽出して埋める。

記録直後のJSON生成時、Review時項目は空欄のままでよい（AIが推測で埋めない）。  
Review時はUpsertではなくシート上で直接追記するか、Review用の変換で該当行を更新する。

### 項目

|項目|必須|記入タイミング|内容|
|---|---|---|---|
|日付|必須|記録時|検証日|
|配置ID|任意|記録時|M00001形式|
|検証ID|任意|記録時|M00001-00001形式|
|配置写真|任意|記録時|画像URL等|
|厚みID|任意|記録時|A00〜A04|
|手球-的球距離|任意|記録時|距離|
|的球-ポケット距離|任意|記録時|距離|
|90度基準ライン|任意|記録時|基準ライン|
|撞点上下|任意|記録時|U1〜U5|
|撞点左右|任意|記録時|L2〜R2|
|力加減ID|任意|記録時|F01〜F05|
|転がりイメージID|任意|記録時|P00〜P55（5刻み。マスタを正とする）|
|撞き方ID|任意|記録時|S01〜S99|
|予測停止位置|任意|記録時|予測|
|実測停止位置|任意|記録時|実測|
|90度からの前進/後退量|任意|記録時|変化量|
|横変化量|任意|記録時|左右変化|
|再現性ID|任意|記録時|R1〜R5。その場で複数回試した結果を記録する|
|気付き|任意|記録時|学び。構造化しない生メモとしてそのまま書く|
|試合で使う判断|任意|Review時|○／△／×。気付きから抽出|
|試合コメント|任意|Review時|判断の補足（近距離なら安定 等）|
|リファレンス候補|任意|Review時|true/false。見返し時に昇格判断する|

## today_focus

対象シート：

- Today's Focus
- Today's Focus履歴

更新方式：

- Today's Focus：Replace
- Today's Focus履歴：Append

### 項目

|項目|必須|内容|
|---|---|---|
|日付|必須|対象日|
|今日のテーマ|必須|テーマ|
|今日意識すること|任意|配列または改行文字列|
|今日の重点課題|任意|配列または改行文字列|
|今日の練習メニュー案|任意|配列|
|メモ|任意|補足|

`総評`・`良かったこと`・`課題`・`振り返り観点`・`明日以降の練習方針`はDEC00016により`today_focus`から外し、`training_records`の`振り返り`行（本ドキュメントのtraining_records参照）へ移した。`today_focus`は次回に向けた前向きな計画情報のみを保持する。

### Today's Focus履歴の保存形式

横持ち1行追記。

```text
日付
今日のテーマ
今日意識すること
今日の重点課題
今日の練習メニュー案
登録日時
```

※ 実シートのヘッダーに`総評`・`良かったこと`・`課題`・`振り返り観点`・`明日以降の練習方針`列が残っている場合でも、DEC00016以降はJSON側から値を送らないため空欄のままとなる（7/12以降のシート再見直しで列自体の扱いを決定する）。

配列は改行区切り文字列に変換する。  
オブジェクト配列は、人が読める文字列またはJSON文字列として保存する。

## エラー時のログ仕様

GASは以下の情報をログに出す。

|項目|内容|
|---|---|
|対象JSONキー|miss_logs等|
|対象シート|ミスログ等|
|更新方式|Append / Upsert / Replace|
|対象index|配列内の番号|
|キー名|ID、課題など|
|キー値|D00001、90度分離など|
|エラー理由|列なし、許可値外、シートなし等|
