# Spreadsheet Design v2.1

## 概要

本ドキュメントは、ビリヤード競技力向上システムで使用するGoogle Spreadsheetの役割と更新ルールを定義する。

現行の正とするSpreadsheetは以下。

```text
9ball_練習記録_260627Update
```

JSON仕様を基準としつつ、現在フェーズでは既存Spreadsheet実態との互換性を最優先する。

---

## 更新方法の定義

| 更新方法 | 意味 |
|---|---|
| Append | 末尾に新規追加する |
| Upsert | 既存行があれば更新し、なければ追加する |
| Replace | 表示用シートの指定範囲を置き換える |
| Manual | GASから自動更新しない |

## メニュー設計

### 目的

次回または当日の練習メニューを設計する。

### 更新方法

Append

### 注意

旧設計資料では `練習設計` と記載されていたが、現行実シート名は `メニュー設計` である。  
GASでは必ず `メニュー設計` を使用する。

## 練習記録_archive

### 目的

既存の練習記録データを保持する（v1.x時代のデータ。Manual化に伴い `練習記録` からリネーム済み）。

### 更新方法

### 注意

`練習ログ` と役割が重複していたため、v2.0でJSONスキーマから除外した。  
既存データは参照用としてシート上に残すが、新規記録は `練習ログ` に一本化する。

## Position Lab

### 目的

90度分離を基準に、厚み・距離・撞点・力加減・撞き方ごとの手球変化を検証する。

### 更新方法

Append

### 位置付け

`Position Lab` は研究ノートであり、確定知識ではない。  
1条件＝1レコードで記録する。

### 主な内容

- 日付
- 配置ID（M00001形式）
- 検証ID（M00001-00001形式）
- 配置写真
- 厚みID（A00〜A04）
- 手球-的球距離
- 的球-ポケット距離
- 90度基準ライン
- 撞点上下（U1〜U5）
- 撞点左右（L2〜R2）
- 力加減ID（F01〜F05）
- 転がりイメージID（P00〜P30）
- 撞き方ID（S01〜S99）
- 予測停止位置
- 実測停止位置
- 90度からの前進/後退量
- 横変化量
- 再現性ID（R1〜R5）
- 気付き
- 試合で使う判断（○／△／×）
- 試合コメント（自由記述）
- リファレンス候補（TRUE/FALSE）

### マスタ参照

| 列 | マスタ | 値範囲 |
|---|---|---|
| 厚みID | 厚みマスタ | A00〜A04 |
| 撞点上下 | 撞点マスタ（上下） | U1〜U5 |
| 撞点左右 | 撞点マスタ（左右） | L2〜R2 |
| 力加減ID | 力加減マスタ | F01〜F05 |
| 転がりイメージID | 転がりイメージマスタ | P00〜P30 |
| 撞き方ID | 撞き方マスタ | S01〜S99 |
| 再現性ID | 再現性マスタ | R1〜R5 |
| 試合で使う判断 | 試合で使う判断マスタ | ○／△／× |

## 変化量マップ_archive

### 目的

旧実験ログ（v1.x時代のデータ。Manual化に伴い `変化量マップ` からリネーム済み）。現行ではPosition Labに近い役割を持つ。

### 更新方法

Manual（system_v2.gsからは書き込まない）

### 方針

v2.0でJSONスキーマから除外した。新規の詳細検証は `Position Lab` に一本化する。

## リファレンス

### 目的

検証を経て、試合中の判断に使える確定知識を保存する。

### 更新方法

Upsert

### キー

ID

### 注意

Reference候補は候補のまま保存してよい。  
GASが自動で正式Reference化しない。

### 列の役割とプルダウン

| 列 | 種類 | プルダウン | 参照先 |
|---|---|---|---|
| カテゴリ | 構造化 | あり | マスタ「リファレンスカテゴリ」列 |
| 厚み | 自由記述 | なし | — |
| 撞点 | 自由記述 | なし | — |
| 力加減 | 自由記述 | なし | — |
| 成功率 | 自由記述 | なし | — |
| 配置ID | 自由入力 | なし | M00001形式 |
| 厚みID | 構造化 | あり | マスタ「厚みID」列 |
| 撞点上下 | 構造化 | あり | マスタ「撞点上下」列 |
| 撞点左右 | 構造化 | あり | マスタ「撞点左右」列 |
| 転がりイメージ | 構造化 | あり | マスタ「転がりイメージID」列 |

自由記述列（厚み・撞点・力加減・成功率）と構造化ID列（厚みID・撞点上下・撞点左右・転がりイメージ）は重複ではなく役割が異なる。  
自由記述列はReference候補段階の人が読む説明用。構造化ID列は絞り込み・集計用。

## Today's Focus

### 目的

当日の練習・試合前に確認する表示用ダッシュボード。

### 更新方法

Replace

### 現行テンプレ構造

```text
Today's Focus
対象日
今日のテーマ
今日意識すること
今日の重点課題
今日の練習メニュー
```

### 注意（項目名について）

表示用テンプレートは「今日の練習メニュー」、Today's Focus履歴は「今日の練習メニュー案」と、末尾の「案」有無が異なる。  
これは別シート・別用途（表示=確定形、履歴=記録時点の案）として意図的に使い分ける。実シート確認時にヘッダー名と一致しているか必ず確認する。

### 注意

固定テンプレートとして扱い、表形式のDBとして扱わない。

## Today's Focus履歴

### 目的

Today's Focusの履歴を保存する。

### 更新方法

Append

### 正しい保存形式

横持ち1行追記。

```text
日付
今日のテーマ
今日意識すること
今日の重点課題
今日の練習メニュー案
総評
良かったこと
課題
振り返り観点
明日以降の練習方針
登録日時
```

### 禁止

Today's Focusの縦型テンプレートをそのまま履歴へ貼り付けない。

## マスタ

### 目的

システム全体で共通利用する入力候補を管理する。

### 現行マスタ列一覧

| 列名 | 用途 |
|---|---|
| コンディション | 練習時の調子 |
| 球種 | 遠厚め、薄めカット、ストップ、バンク、コンビ等（ショットを特徴付ける最もわかりやすいラベル） |
| 分類 | シュート、出し、判断、セーフティ等 |
| 選択 | 攻めた、守った、迷った |
| 原因 | 技術、判断、ライン読み、コンディション、その他 |
| 成否 | 成功、失敗 |
| 改善状況 | 未着手〜アーカイブ候補 |
| 分類詳細 | 厚み、ストローク、撞点、ライン読み、力加減、配置判断、ルート選択、セーフティ精度、距離感、その他 |
| 親分類 | シュート、出し、判断、セーフティ等 |
| メニュー例 | 練習メニュー例 |
| 課題タグ | 課題管理シートとの紐付け（課題管理A列「課題」と一致させる） |
| テーマタグ | 学習テーマ・競技原則・判断軸 |
| 力加減 | F01〜F05（Position Lab用） |
| 撞き方 | S01〜S99（Position Lab用） |
| 再現性 | R1〜R5（Position Lab用） |
| 試合で使う判断 | ○／△／×（Position Lab・Pro Reference用） |
| 厚みID | A00〜A04 |
| 撞点上下 | U1〜U5 |
| 撞点左右 | L2〜R2 |
| 転がりイメージID | P00〜P30 |
| 決定事項カテゴリ | 練習原則、ショットルール、判断基準、記録方針、試合方針、Position Lab、試合判断、メンタル |
| 習慣化 | 練習中、定着、保留、廃止 |
| リファレンスカテゴリ | 変化量マップ、Reference候補 |
| 優先度 | 1〜5（課題管理・決定事項共通） |
| 有効 | TRUE／FALSE（決定事項用） |

### 方針

プルダウン候補はマスタまたは既存入力規則に合わせる。  
GASで入力規則を削除しない。

## 実装上の保護ルール

- `clearDataValidations()` 禁止
- 数式列への `setValues()` 禁止
- シート全体 `clear()` 禁止
- ヘッダー行の勝手な上書き禁止
- 実シート名以外の使用禁止
