# Data Model v1.1

## 目的

本ドキュメントは、ビリヤード競技力向上システムで使用する共通マスタ・ID体系を定義する。

JSON、GAS、Spreadsheet、将来のFlutterアプリで同じID体系を利用する。

---

## 設計方針

- マスタは最小構成から始める
- 運用で頻出する概念のみ追加する
- 既存IDの意味は変更しない
- 集計したい項目だけをマスタ化する
- 狙い・学び・次回イメージ・メモなど、毎回内容が変わるものは自由入力とする
- 既存Spreadsheetの入力規則・マスタ値を優先する

---

## 現行マスタシート

現行 `マスタ` シートには以下の列が存在する。

| 列 | 用途 |
|---|---|
| コンディション | 練習時の調子 |
| 球種 | 遠厚め、遠薄め、長距離、ストップ、バンク、コンビなど（ショットを特徴付ける最もわかりやすいラベル） |
| 分類 | シュート、出し、判断、セーフティなど |
| 選択 | 攻めた、守った、迷った |
| 原因 | 技術、判断、ライン読み、コンディション等 |
| 成否 | 成功、失敗 |
| 改善状況 | 未着手、練習中、改善中、ほぼ解決、解決等 |
| 分類詳細 | 厚み、ストローク、撞点、ライン読み、力加減、配置判断、ルート選択、セーフティ精度、距離感、その他（技術・判断・セーフティ横断の詳細分類） |
| 親分類 | シュート、出し、判断、セーフティ等 |
| メニュー例 | 練習メニュー例 |
| 課題タグ | 課題タグ |
| テーマタグ | 学習テーマ・競技原則・判断軸 |
| 力加減 | F01〜F05（Position Lab用） |
| 撞き方 | S01〜S99（Position Lab用） |
| 再現性 | R1〜R5（Position Lab用） |
| 試合で使う判断 | ○／△／×（Position Lab・Pro Reference用） |
| 厚みID | A00〜A04 |
| 撞点上下 | U1〜U5 |
| 撞点左右 | L2〜R2 |
| 転がりイメージID | P00〜P30 |
| 決定事項カテゴリ | 練習原則、ショットルール、判断基準、記録方針、試合方針、Position Lab、試合判断、メンタル |
| 習慣化 | 練習中、定着、保留、廃止 |
| リファレンスカテゴリ | 変化量マップ、Reference候補 |
| 優先度 | 1〜5（課題管理・決定事項共通） |
| 有効 | TRUE／FALSE（決定事項用） |

GASは入力規則を壊さず、既存候補に合う値を出力する。

## 厚みマスタ

Position Labでは、当面IDと自然言語の両方を許容する。  
ID管理する場合は以下を使用する。

| ID | 名称 | 説明 |
|---|---|---|
| A00 | 全厚 | 100%ヒット |
| A01 | 3/4 | 厚め |
| A02 | 半球 | 50%・基準厚み |
| A03 | 1/4 | 薄め |
| A04 | 薄い | 1/8前後、または極薄寄り |

必要になった場合のみ、A05以降を追加する。

## 転がりイメージマスタ

転がりイメージは、力加減そのものではなく、試合中に再現したい手球の転がり量を表す。

| ID | 転がりイメージ |
|---|---|
| P00 | ストップ |
| P05 | 0.5ポイント |
| P10 | 1.0ポイント |
| P15 | 1.5ポイント |
| P20 | 2.0ポイント |
| P25 | 2.5ポイント |
| P30 | 3.0ポイント |

球1個分、球2個分などの短距離表現は、まず自由入力で運用する。頻出する場合は、P01、P02などとして後から追加する。

|---|---|

## 試合で使う判断マスタ

Position Lab専用。検証結果の実戦投入可否を3段階で評価する。

| 記号 | 意味 |
|---|---|
| ○ | 使える |
| △ | 要検証 |
| × | 再現性低い・使わない |

`試合コメント` 列に補足を自由記述する（例：近距離・2/3厚なら安定）。

## ID体系

### 配置ID

形式：`M00001`

### 検証ID

形式：`M00001-00001`

### リファレンスID

形式：`REF00001`

### 決定事項ID

新規登録形式：`D00001`（必須）

既存データには `D001`〜`D022` 形式が残っている。  
既存IDは変更しないが、新規登録は必ず `D00001` 形式（5桁ゼロ埋め）を使用する。  
ID列内に旧形式（`D001`）と新形式（`D00001`）が混在する状態を許容する。GASのUpsert照合は文字列完全一致のため、形式混在による不具合はない。

### Pro ReferenceID

形式：`PR00001`

Pro ReferenceをUpsert対象にするための新設キー。既存シートにID列がない場合は、実シート確認時に列追加する。

### 課題ID

将来形式：`ISS00001`

現行v1.1では `課題` 名をキーとして運用する。

## 正規化ルール

GASでUpsertキーを照合する場合、最低限以下を行う。

```text
前後空白の除去
全角スペースの半角化
連続スペースの圧縮
```

課題名の意味が同じでも表記が異なる場合は別課題になりうるため、ChatGPT出力時は既存課題名を優先する。

---

## Reference候補

`Reference候補` は正式リファレンスではなく、検証待ちの状態を示す。

GASは以下を行わない。

- 候補を正式カテゴリへ自動変換する
- 不足項目を推測で埋める
- Position Lab未検証の内容を確定知識として登録する
