# JSON仕様 v1.0

## 位置付け

本ドキュメントは、ビリヤード競技力向上システムにおける JSON データ仕様を定義する。

JSON は、以下すべての共通データ形式とする。

```text
ChatGPT
  ↓ JSON
GAS
  ↓
Google Spreadsheet

将来:
Flutter
  ↓ JSON
GAS / API
  ↓
Google Spreadsheet または DB
```

スプレッドシートは JSON の保存・表示先であり、将来的な Flutter / DB 移行時も JSON 仕様を中心に設計する。

---

## バージョン方針

- 現行仕様は `schema_version: "1.0"` とする。
- v1.x では既存キー・既存IDの意味を変更しない。
- 新しい項目やシートを追加する場合は、既存JSONを壊さない形で追加する。
- 既存キー名の変更が必要な場合は v2.0 として扱う。
- マスタ項目は最小構成で開始し、運用上頻出する概念のみ追加する。

---

## 基本方針

### 1. JSONキー名はスプレッドシート列名と一致させる

例：

```json
{
  "日付": "2026/06/29",
  "厚みID": "A02",
  "転がりイメージ": "P15"
}
```

GASはシート1行目のヘッダーとJSONキー名を照合して書き込む。

### 2. 1シート1役割

各シートの役割を混ぜない。

- ミスログ：起きた事象の分析
- 課題管理：継続課題
- 決定事項：競技原則
- 変化量マップ：実験ノート
- リファレンス：確定知識

### 3. マスタ化するのは集計したい項目だけ

マスタ化するもの：

- 厚み
- 撞点
- 転がりイメージ
- 評価

自由入力にするもの：

- 狙い
- 実測
- 次回イメージ
- 学び
- メモ

### 4. AIで検索・要約できるものは無理にマスタ化しない

狙い分類などは v1.0 では自由入力とする。

## run

`run` は、GASでどの更新処理を実行するかを指定する。

```json
"run": {
  "mist_logs": true,
  "issue_updates": false
}
```

- `true`：処理する
- `false`：処理しない
- キーが存在しない場合は `false` 扱いを推奨

---

## 更新方式

|方式|意味|
|---|---|
|Append|末尾に追加|
|Update|既存行を更新|
|Upsert|既存行があれば更新、なければ追加|
|Replace|表示用シートを置き換え|

# 各配列仕様

## menu_plans

対象シート：練習設計  
更新方式：Append

練習前に作る次回練習メニューを記録する。

### 項目

|項目|必須|内容|
|---|---|---|
|日付|必須|対象日|
|メニュー名|必須|練習メニュー名|
|時間|任意|予定時間|
|目的|任意|なぜやるか|
|内容|任意|具体的な練習内容|
|重点課題|任意|関連課題|
|メモ|任意|補足|


### 例

```json
{
  "日付": "2026/06/29",
  "メニュー名": "変化量マップ作成",
  "時間": "30分",
  "目的": "90度分離を起点に転がりイメージと実測の差を確認する",
  "内容": "M00001でA02、U4、C0、P05〜P20を検証",
  "重点課題": "変化量マップ",
  "メモ": ""
}
```

## issue_updates

対象シート：課題管理  
更新方式：Upsert  
キー：課題

既存課題があれば更新し、なければ追加する。

### キー照合ルール

- v1.0では `課題` をUpsertキーとして扱う。
- GAS側では照合前に、前後空白・全角/半角スペースを正規化する。
- 課題名の意味が同じでも表記が異なる場合は別課題として扱われる可能性があるため、ChatGPT出力時は既存課題名を優先する。
- 課題ID（ISS00001形式）は将来検討事項とし、v1.0では必須にしない。

### 項目

|項目|必須|内容|
|---|---|---|
|課題|必須|課題名|
|原因|任意|主原因|
|課題と練習方法|任意|練習内容|
|改善状況|任意|未着手、練習中、改善中、ほぼ解決、解決|
|優先度|任意|数値または文字列|
|メモ|任意|補足|

### 改善状況の許可値

- 未着手
- 練習中
- 改善中
- ほぼ解決
- 解決

## variation_map_records

対象シート：変化量マップ  
更新方式：Append

配置ごとの実験ノート。

### 項目

|項目|必須|内容|
|---|---|---|

## today_focus

対象シート：

- Today's Focus
- Today's Focus履歴

更新方式：

- Today's Focus：Replace
- Today's Focus履歴：Append

その日に見るための要点表示。  
履歴を残すことで、週次・月次で集中テーマをAI集計できるようにする。

### 項目

|項目|必須|内容|
|---|---|---|
|日付|必須|対象日|
|今日のテーマ|必須|テーマ|
|今日意識すること|任意|配列または改行文字列|
|重点課題|任意|配列または改行文字列|
|練習メニュー|任意|配列|
|振り返り観点|任意|配列|
|メモ|任意|補足|

### スプレッドシート保存形式

- 配列フィールドは、改行区切りテキスト（`\n` 結合）としてセルに保存する。
- オブジェクト配列（例：`練習メニュー`）は、v1.0ではJSON文字列としてセルに保存する。
- 理由：Today's Focus は人が読む表示シートであり、v1.0では視認性を優先するため。

### 例

```json
{
  "日付": "2026/06/29",
  "今日のテーマ": "90度分離を起点に、変化量マップを作る",
  "今日意識すること": [
    "近い球で厚みと90度分離をセットで確認する",
    "転がりイメージと実測の差を見る",
    "試合で外した配置を再現する"
  ],
  "重点課題": [
    "変化量マップ",
    "90度分離",
    "薄めカット"
  ],
  "練習メニュー": [
    {
      "メニュー名": "近距離90度分離確認",
      "時間": "20分",
      "目的": "厚みと90度分離をセットで確認"
    }
  ],
  "振り返り観点": [
    "90度予測は合っていたか",
    "転がりイメージと実測は一致したか"
  ],
  "メモ": ""
}
```

|ID|名称|定義|
|---|---|---|
|U1|真下|下端から半タップ以内|

## 撞点マスタ：左右

|ID|名称|定義|
|---|---|---|
|L2|真左|左端から半タップ以内|
|L1|左|中心から半タップ左|
|C0|中心|左右回転なし|
|R1|右|中心から半タップ右|
|R2|真右|右端から半タップ以内|

## 転がりイメージマスタ

|ID|名称|
|---|---|
|P00|ストップ|

|ID|名称|
|---|---|

# ID仕様

## 配置ID

形式：

```text
M00001
```

例：

```text
M00001
M00002
M01000
```

## 検証ID

形式：

```text
M00001-00001
```

例：

```text
M00001-00001
M00001-00002
M00002-00001
```

## 決定事項ID

形式：

```text
D00001
```

## リファレンスID

形式：

```text
REF00001
```

## 課題ID

将来的には以下形式を採用可能。

```text
ISS00001
```

v1.0では課題名をキーとして運用する。

# エラー時のログ仕様

GASは以下の情報をログに出す。

|項目|内容|
|---|---|
|対象JSONキー|mist_logs等|
|対象シート|ミスログ等|
|更新方式|Append / Upsert / Replace|
|対象index|配列内の番号|
|キー名|ID、課題など|
|キー値|D00001、90度分離など|
|エラー理由|列なし、許可値外、シートなし等|

例：

```text
[ERROR] issue_updates[2]
sheet=課題管理
mode=Upsert
key=課題
value=変化量マップ
reason=改善状況「新規」は許可値外
```

---

# 互換性ルール

- 既存IDの意味は変更しない。
- 既存JSONキー名は変更しない。
- 新しい項目は追加のみ。
- 廃止項目は即削除せず、GAS側で無視できるようにする。
- v1.xでは既存データが処理不能になる変更を行わない。
