# Data Model v1.0

## 目的

本ドキュメントは、ビリヤード競技力向上システムで使用する共通マスタ・ID体系を定義する。

JSON、GAS、Spreadsheet、将来のFlutterアプリで同じID体系を利用する。

---

## 設計方針

- マスタは最小構成で開始する
- 運用で頻出する概念のみ追加する
- 既存IDの意味は変更しない
- 集計したい項目だけをマスタ化する
- 狙い・学び・次回イメージなど、毎回内容が変わるものは自由入力とする

---

## 厚みマスタ

| ID | 名称 | 説明 |
|---|---|---|
| A00 | 全厚 | 100%ヒット |
| A01 | 3/4 | 厚め |
| A02 | 半球 | 50%・基準厚み |
| A03 | 1/4 | 薄め |
| A04 | 薄い | 1/8前後、または極薄寄り |

必要になった場合のみ、A05以降を追加する。

| ID | 名称 | 定義 |

| ID | 名称 | 定義 |
|---|---|---|
| L2 | 真左 | 左端から半タップ以内 |
| L1 | 左 | 中心から半タップ左 |

## 転がりイメージマスタ

転がりイメージは、力加減そのものではなく、試合中に再現したい手球の転がり量を表す。

| ID | 転がりイメージ |
|---|---|
| P00 | ストップ |
| P05 | 0.5ポイント |
| P10 | 1.0ポイント |
| P15 | 1.5ポイント |
| P20 | 2.0ポイント |
| P25 | 2.5ポイント |
| P30 | 3.0ポイント |

球1個分、球2個分などの短距離表現は、まず自由入力で運用する。頻出する場合は、P01、P02などとして後から追加する。

## 評価マスタ

| 記号 | 意味 |
|---|---|
| ◎ | 狙い通り |
| ○ | 概ね成功 |
| △ | 改善余地あり |

## ID体系

### 配置ID

形式：`M00001`

例：

- M00001
- M00002
- M01000

---

### 検証ID

形式：`M00001-00001`

例：

- M00001-00001
- M00001-00002
- M00002-00001

---

### リファレンスID

形式：`REF00001`

例：

- REF00001
- REF00002
- REF01000

---

### 決定事項ID

形式：`D00001`

例：

- D00001
- D00002
- D01000

既存データにD014などがある場合、移行時にD00014へ統一する。

---

### 課題ID

形式：`ISS00001`

例：

- ISS00001
- ISS00002
- ISS01000

v1.0では課題名をキーとして管理してもよい。将来的には課題ID管理へ移行する。

## 変化量マップでの利用例

| 項目 | 例 |
|---|---|
| 検証ID | M00001-00001 |
| 配置ID | M00001 |
| 厚みID | A02 |
| 撞点上下 | U4 |
| 撞点左右 | R1 |
| 転がりイメージ | P15 |
| 狙い | 3サイド左振りへの出し。90度分離1クッション。球2個分転がり。 |
| 実測 | 球4個分ロング |
| 次回イメージ | P15では出すぎ。次回はP12相当を試す。 |
| 評価 | × |
| 学び | この角度は90度分離を手前に誤認しやすい。 |
| リファレンス候補 | TRUE |
