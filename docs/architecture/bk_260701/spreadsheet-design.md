# Spreadsheet Design v1.0

## 概要

本ドキュメントは、ビリヤード競技力向上システムで使用する Google Spreadsheet の役割と更新ルールを定義する。

JSON仕様を基準とし、各シートは原則として1つの役割のみを持つ。

---

## 更新方法の定義

| 更新方法 | 意味 |
|---|---|
| Append | 末尾に新規追加する |
| Update | 既存行を探して更新する |
| Upsert | 既存行があれば更新し、なければ追加する |
| Replace | 表示用シートを全体更新する |

---

## シート一覧

| JSONキー | 対象シート | 更新方法 | キー | 役割 |
|---|---|---|---|---|
| menu_plans | 練習設計 | Append | なし | 次回練習メニューの計画 |
| training_records | 練習記録 | Append | なし | 実施した練習内容の記録 |
| match_reviews | 試合振り返り | Append | なし | 試合単位の総括 |
| mist_logs | ミスログ | Append | なし | 1ショット単位のミス・成功・気付き |
| issue_updates | 課題管理 | Upsert | 課題 | 継続課題の管理 |
| decision_updates | 決定事項 | Upsert | ID | 競技原則・判断基準の管理 |
| variation_map_records | 変化量マップ | Append | なし | 配置実験ログ |
| reference_updates | リファレンス | Upsert | ID | 確定知識の管理 |
| today_focus | Today's Focus | Replace | なし | 当日表示用ダッシュボード |
| today_focus | Today's Focus履歴 | Append | なし | Today's Focusの履歴保存 |
| - | マスタ | 手動更新 | - | 共通ID・選択肢の管理 |

## 練習設計

### 目的

次回または当日の練習メニューを設計する。

### 注意点

既存の練習設計シートには、以下の2種類の情報が混在しやすい。

- 練習前のメニュー計画
- 練習後の総評・振り返り・重点課題

v1.0では既存シートを活かしつつ、JSONでは `menu_plans` として練習前の計画情報を扱う。

練習後の実績・振り返りは `training_records` 側で扱う。

### 更新方法

Append

### 目的

実際に行った練習内容を記録する。

### 更新方法

### 主な内容

- メニュー名
- 時間
- 目的
- メモ

## 試合振り返り

### 目的

試合単位で総括する。

### 更新方法

Append

### 主な内容

- 日付
- 大会名/練習試合
- 形式
- 結果
- 良かったこと
- 失点原因
- 判断ミス
- 技術ミス
- テーマ実行度
- 今日の学び
- 次回改善
- 課題候補
- 決定事項候補

### 備考

横長テーブルは見づらくなるため、将来的にブロック表示用のビューまたは別シートを検討する。

## ミスログ

### 目的

1ショット単位で、ミス・成功・気付き・フィードバックを記録する。

### 更新方法

Append

### 備考

ミスログは原因分析が目的であり、配置ごとの実験結果は変化量マップに記録する。

---

## 課題管理

### 目的

継続的に改善する課題を管理する。

### 更新方法

Upsert

### キー

課題

### 注意点

既存の課題名があれば更新する。  
存在しない課題名であれば新規追加する。

---

## 決定事項

### 目的

競技中に迷わないための原則・判断基準を管理する。

### 更新方法

Upsert

### キー

ID

### ID形式

D00001 形式を基本とする。

---

## 変化量マップ

### 目的

配置ごとの実験結果を蓄積する。

リファレンスではなく、実験ノートとして扱う。

### 更新方法

Append

### 主な内容

- 検証ID
- 日付
- 配置ID
- 配置写真
- 厚みID
- 撞点上下
- 撞点左右
- 転がりイメージ
- 狙い
- 実測
- 次回イメージ
- 評価
- 学び
- リファレンス候補

### 備考

同じ配置でも、厚み・撞点・転がりイメージが違えば別行として記録する。

---

## リファレンス

### 目的

検証を経て、試合中の判断に使える確定知識を保存する。

### 更新方法

Upsert

### キー

ID

### ID形式

REF00001 形式を基本とする。

### 主な内容

- ID
- カテゴリ
- テーマ
- 配置名
- 配置写真
- 厚みID
- 撞点上下
- 撞点左右
- 転がりイメージ
- 実測
- 成功率
- 気付き
- 次回検証
- 登録日

### 備考

変化量マップで検証した内容のうち、再現性があり、試合判断に使えるものを昇格させる。
厚み・撞点・転がりイメージは、変化量マップと同じID体系を使う。

---

## Today's Focus

### 目的

当日の練習・試合前に確認する表示用ダッシュボード。

### 更新方法

Replace

### 保存形式

- 配列フィールドは改行区切りテキストとして保存する。
- オブジェクト配列はJSON文字列として保存する。

### 備考

Today's Focusは蓄積用ではなく、当日表示用とする。  
履歴は `Today's Focus履歴` に保存する。

---

## Today's Focus履歴

### 目的

Today's Focusの履歴を保存する。

### 更新方法

Append

### 主な内容

- 日付
- 今日のテーマ
- 今日意識すること
- 重点課題
- 練習メニュー
- 振り返り観点

### 備考

後日AIで以下を整理できるようにする。

- 今週の集中テーマ
- 繰り返し出ている重点課題
- 試合前に意識していたこと
- 改善が進んでいない課題
- 来週の練習テーマ候補

---

## マスタ

### 目的

システム全体で共通利用するID・選択肢を管理する。

### 主なマスタ

- 厚みマスタ
- 撞点マスタ
- 転がりイメージマスタ
- 評価

### 方針

マスタは最小構成で開始する。  
運用の中で頻出する概念のみ追加する。  
既存IDの意味は変更しない。
