## 取り切りルート提案（ネタ・未実装）

配置図（球位置・キューボール）を入力に、**取り切り順とライン案**を候補として出す機能。

### コンセプト

- 「正解を1つ計算」ではなく **候補シーケンス＋ライン案** を提示する補助ツール
- 例: **2 → 4 → 5** の順で取る場合、各手の
  - キュー → オブジェクト
  - オブジェクト → ポケット
  のラインをセットで表示
- 候補は **2〜3本**（A案: 直球 / B案: 1クッション など）

### 見せ方のイメージ

- **3球ごとのブロック**で、**2〜3手先**までを提案
  - ブロック1（1〜3球目）: 候補A `3→7→5` / 候補B `7→3→5`
  - ブロック2（4〜6球目）: …
  - 残り3球以下ならブロックを短くする
- タップで **1手目のラインだけ**エディタ上に重ね表示（既存 `TrajectoryGeometry` と連携）
- 各候補に **1行理由**（障害が少ない / 次の球が近い など）

### 既存資産（土台）

- `BallLayoutEditorScreen`: 球の正規化座標、ポケット位置
- `TrajectoryGeometry`: 1ショット分の軌道・クッション反射
- 手動軌道編集モード（`_trajMode`）

### v1 でやること（最小）

1. 各オブジェクト球について入りやすいポケットをスコア化
2. 通り道に他球がないか（簡易当たり判定）
3. 2〜3手先までの候補シーケンスを **貪欲法 or ビームサーチ（浅い）** で生成
4. 上位2〜3シーケンスを UI にリスト表示 → 選択でライン描画

### v1 のスコア要素（案）

| 要素 | 内容 |
|------|------|
| 通れるか | 他球・クッション障害 |
| 入りやすさ | 角度・距離 |
| つながり | 次の球へのキュー位置（2〜3手目） |

### 難しい / 後回し

- スピン・キック・クラッチ・玉回転
- 実戦レベルのポジション最適化
- 唯一の正解データ（存在しない前提で設計）

### 実地テスト後に詰めること

- 9球専用か汎用（10球・15球）か
- 検出配置のズレがライン提案に与える許容誤差
- ブロックサイズ（3球固定 vs 可変）

### 参考フロー（完成イメージ）

```
配置を取る → 図にする → 候補シーケンス表示 → ライン確認 → 手動微調整
```
